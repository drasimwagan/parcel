04 HTTP layer — router, shell_api, templates

Time to write real routes. We'll add list, create, detail, and delete handlers to the Notes module, each one mediated by the SDK's shell_api facade. This is where the discipline of "modules import parcel_sdk, never parcel_shell" starts showing its teeth.

The shell_api facade — one page of reference

Every function your router will call, in one list:

  • shell_api.get_session() — returns the FastAPI dep that yields an AsyncSession. Use inside Depends(...). Commits on successful completion, rolls back on exception. You never call db.commit() yourself in a handler.
  • shell_api.require_permission(name) — returns a FastAPI dep that (a) requires an authenticated user and (b) enforces the named permission. On unauthenticated → 303 to /login?next=…. On missing permission → 303 to / with an error flash.
  • shell_api.effective_permissions(request, user) — returns the full set of permission names the user currently has. You use this to filter the composed sidebar.
  • shell_api.get_templates() — returns the shared Jinja2Templates. Your module's templates_dir was prepended to the loader when you were installed, so you reference templates by their path under your dir ("notes/index.html").
  • shell_api.sidebar_for(request, perms) — returns the composed sidebar (shell sections + every active module's section) filtered by the user's perms. Pass to your template context.
  • shell_api.set_flash(response, flash) — attach a one-shot banner to the next page render. flash is a Flash(kind, msg) where kind is "success", "error", or "info".

A shared context helper

Every HTML route needs the same four keys for the base template: user, sidebar, active_path, settings. Factor this into a small helper at the top of router.py: 

async def _ctx(request: Request, user: Any, path: str) -> dict[str, Any]:
    perms = await shell_api.effective_permissions(request, user)
    return {
        "user": user,
        "sidebar": shell_api.sidebar_for(request, perms),
        "active_path": path,
        "settings": request.app.state.settings,
    }

The full router

Replace src/parcel_mod_notes/router.py: 

from __future__ import annotations

import uuid
from typing import Any

from fastapi import APIRouter, Depends, Form, HTTPException, Request, Response, status
from sqlalchemy.ext.asyncio import AsyncSession
from starlette.responses import HTMLResponse, RedirectResponse

from parcel_mod_notes import service
from parcel_sdk import shell_api
from parcel_sdk.shell_api import Flash

router = APIRouter(tags=["mod-notes"])


async def _ctx(request: Request, user: Any, path: str) -> dict[str, Any]:
    perms = await shell_api.effective_permissions(request, user)
    return {
        "user": user,
        "sidebar": shell_api.sidebar_for(request, perms),
        "active_path": path,
        "settings": request.app.state.settings,
    }


@router.get("/", response_class=HTMLResponse)
async def notes_list(
    request: Request,
    user: Any = Depends(shell_api.require_permission("notes.read")),
    db: AsyncSession = Depends(shell_api.get_session()),
) -> Response:
    notes = await service.list_notes(db)
    tpl = shell_api.get_templates()
    return tpl.TemplateResponse(
        request,
        "notes/list.html",
        {**(await _ctx(request, user, "/mod/notes")), "notes": notes},
    )


@router.get("/new", response_class=HTMLResponse)
async def notes_new_form(
    request: Request,
    user: Any = Depends(shell_api.require_permission("notes.write")),
    db: AsyncSession = Depends(shell_api.get_session()),
) -> Response:
    tpl = shell_api.get_templates()
    return tpl.TemplateResponse(
        request,
        "notes/new.html",
        await _ctx(request, user, "/mod/notes"),
    )


@router.post("/")
async def notes_create(
    request: Request,
    title: str = Form(...),
    body: str = Form(""),
    user: Any = Depends(shell_api.require_permission("notes.write")),
    db: AsyncSession = Depends(shell_api.get_session()),
) -> Response:
    if not title.strip():
        response = RedirectResponse(url="/mod/notes/new", status_code=303)
        shell_api.set_flash(response, Flash(kind="error", msg="Title is required."))
        return response
    note = await service.create_note(db, title=title, body=body)
    response = RedirectResponse(url=f"/mod/notes/{note.id}", status_code=303)
    shell_api.set_flash(response, Flash(kind="success", msg="Note created."))
    return response


@router.get("/{note_id}", response_class=HTMLResponse)
async def notes_detail(
    note_id: uuid.UUID,
    request: Request,
    user: Any = Depends(shell_api.require_permission("notes.read")),
    db: AsyncSession = Depends(shell_api.get_session()),
) -> Response:
    note = await service.get_note(db, note_id)
    if note is None:
        raise HTTPException(status.HTTP_404_NOT_FOUND, "note_not_found")
    tpl = shell_api.get_templates()
    return tpl.TemplateResponse(
        request,
        "notes/detail.html",
        {**(await _ctx(request, user, "/mod/notes")), "note": note},
    )


@router.post("/{note_id}/delete")
async def notes_delete(
    note_id: uuid.UUID,
    request: Request,
    user: Any = Depends(shell_api.require_permission("notes.write")),
    db: AsyncSession = Depends(shell_api.get_session()),
) -> Response:
    note = await service.get_note(db, note_id)
    if note is None:
        raise HTTPException(status.HTTP_404_NOT_FOUND, "note_not_found")
    await service.delete_note(db, note=note)
    response = RedirectResponse(url="/mod/notes/", status_code=303)
    shell_api.set_flash(response, Flash(kind="info", msg="Note deleted."))
    return response

Five handlers, no shell imports, no raw SQL. Every HTTP concern — auth, permissions, database sessions, templates, flashes — goes through the facade. If you ever find yourself tempted to from parcel_shell import something, stop. Either the thing you need belongs in your own module's service layer, or it belongs in a future expansion of the SDK.

Why the odd HTML escaping above? The tutorial pages are themselves HTML, so Jinja syntax needs to be shown as entities. The files you actually write use real angle brackets. Look at modules/contacts/src/parcel_mod_contacts/templates/contacts/ in the repo for unescaped examples.

Route-order note

FastAPI matches routes in declaration order. If you later add a /{note_id} route using a UUID type converter, and then a literal-path route like /archive, the UUID route will return 422 for any non-UUID path that reaches it first. Declare literal paths before parameterised paths, or use path-prefix guards like /notes/{note_id}. This bit us in the Contacts module — see the comment in modules/contacts/src/parcel_mod_contacts/router.py.
