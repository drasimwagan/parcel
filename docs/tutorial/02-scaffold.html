02 Scaffolding with the CLI

The parcel CLI ships a scaffolder. One command produces a gate-passing, end-to-end working skeleton. We'll walk through every file it generates so nothing is mysterious later when we start editing.

Run it

uv run parcel new-module notes
uv sync --all-packages

The first command writes modules/notes/ with the skeleton. The second makes the new package importable (the uv workspace picks it up automatically because modules/* is already a member in the root pyproject.toml).

Naming. Module names must be snake_case and match [a-z][a-z0-9_]*. That becomes your package name (parcel_mod_notes), your schema name (mod_notes), your URL prefix (/mod/notes/), and the prefix of your permissions (notes.read, notes.write).

The generated tree

modules/notes/
├── pyproject.toml # entry point + deps
├── README.md
├── src/parcel_mod_notes/
│ ├── __init__.py # Module(...) manifest
│ ├── models.py # SQLAlchemy DeclarativeBase
│ ├── router.py # FastAPI APIRouter
│ ├── alembic.ini # Alembic pointer
│ ├── alembic/
│ │ ├── env.py # delegates to parcel_sdk
│ │ ├── script.py.mako
│ │ └── versions/
│ │ └── 0001_init.py # CREATE SCHEMA mod_notes
│ └── templates/notes/
│ └── index.html # Jinja template
└── tests/
   └── test_smoke.py # identity assertion

Ten files. Every real module ends up in this shape — add more models, more routes, more templates, but the tree stays recognisable.

File-by-file

pyproject.toml

[project]
name = "parcel-mod-notes"
version = "0.1.0"
description = "Parcel module: notes"
readme = "README.md"
requires-python = ">=3.12"
license = { text = "MIT" }
dependencies = [
    "parcel-sdk",
    "fastapi>=0.115",
]

[project.entry-points."parcel.modules"]
notes = "parcel_mod_notes:module"

[tool.uv.sources]
parcel-sdk = { workspace = true }

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[tool.hatch.build.targets.wheel]
packages = ["src/parcel_mod_notes"]

The runtime dep list is intentionally tiny — parcel-sdk and fastapi. Nothing else. That's the whole point of the SDK facade from Phase 6: your module doesn't pull in the shell or any of its deps as a runtime requirement. Your tests will need parcel-shell, but that goes into a dev-only group (we'll see this in the testing step).

src/parcel_mod_notes/__init__.py

from __future__ import annotations

from pathlib import Path

from parcel_mod_notes.models import metadata
from parcel_mod_notes.router import router
from parcel_sdk import Module

module = Module(
    name="notes",
    version="0.1.0",
    permissions=(),
    capabilities=(),
    alembic_ini=Path(__file__).parent / "alembic.ini",
    metadata=metadata,
    router=router,
    templates_dir=Path(__file__).parent / "templates",
    sidebar_items=(),
)

__all__ = ["module"]

This is the single most important file in your module. It's the manifest the shell reads. Over the next few steps we'll fill in the empty tuples (permissions, sidebar_items) and wire in the real entity model.

src/parcel_mod_notes/models.py

from __future__ import annotations

from sqlalchemy import MetaData
from sqlalchemy.orm import DeclarativeBase

metadata = MetaData(schema="mod_notes")


class NotesBase(DeclarativeBase):
    metadata = metadata  # type: ignore[assignment]

Every table class will inherit NotesBase. The MetaData(schema="mod_notes") line is what pins every table to your schema — you won't repeat the schema name on each class.

src/parcel_mod_notes/router.py

from __future__ import annotations

from typing import Any

from fastapi import APIRouter, Depends, Request, Response
from starlette.responses import HTMLResponse

from parcel_sdk import shell_api

router = APIRouter(tags=["mod-notes"])


@router.get("/", response_class=HTMLResponse)
async def index(
    request: Request,
    user: Any = Depends(shell_api.require_permission("")),
) -> Response:
    # Replace "" with a real permission once you declare one in
    # module.permissions. Until then, only authenticated users pass.
    tpl = shell_api.get_templates()
    perms = await shell_api.effective_permissions(request, user)
    return tpl.TemplateResponse(
        request,
        "notes/index.html",
        {
            "user": user,
            "sidebar": shell_api.sidebar_for(request, perms),
            "active_path": "/mod/notes",
            "settings": request.app.state.settings,
        },
    )

A single placeholder route with an intentionally-incomplete require_permission(""). That's a TODO marker — we'll replace it with notes.read once we declare that permission.

Alembic scaffolding

alembic.ini + alembic/env.py + alembic/script.py.mako + a first 0001_init.py revision that creates the schema. You'll rarely touch env.py — it simply imports your module's manifest and calls parcel_sdk.alembic_env.run_async_migrations(module). The rest of the Alembic machinery is wired the way you'd expect.

templates/notes/index.html

{% extends "_base.html" %}
{% block content %}
<h1>notes</h1>
<p>Hello from the notes module.</p>
{% endblock %}

_base.html is the shell's base layout — topbar, sidebar, flash region, theme switching. You extend it; you don't rebuild it.

tests/test_smoke.py

from __future__ import annotations

from parcel_mod_notes import module


def test_module_identity() -> None:
    assert module.name == "notes"
    assert module.version == "0.1.0"

A ten-line smoke test. It imports your manifest and asserts the name. That's enough to catch "I broke __init__.py in a way that doesn't import" — useful on every Phase 6+ CLI run.

Verify the scaffold works

uv run pytest modules/notes/

The smoke test should pass. Also run: 

uv run parcel sandbox install ./modules/notes

That installs your (empty) module into a sandbox. Either the gate says "pass" and you get a sandbox UUID, or it gives you a focused error message. A fresh scaffold passes the gate with zero warnings — if it doesn't, something is genuinely wrong.

Rule of thumb. Commit after scaffolding. Before you start changing anything substantial, the gate-clean scaffold is your known-good baseline. Every edit is a diff from that baseline.
